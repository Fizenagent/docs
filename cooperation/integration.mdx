## QR payment details

QR payments are the fastest way to accept USDT at a physical location. You can use:

- **Static QR** — a single QR code tied to your merchant wallet. Display it at your counter or print it. The customer enters the amount manually.
- **Dynamic QR** — a unique QR code generated per transaction via the dashboard or API. The amount is embedded in the QR, so the customer just scans and confirms.

<Tip>
  Dynamic QR codes reduce errors and speed up checkout. Use them if your point-of-sale system can make API calls or if you manage orders through a dashboard.
</Tip>

## API integration details

The Fizen Pay API lets you build crypto payment acceptance directly into your checkout flow. Key capabilities include:

- Create payment requests with a specified USDT amount and reference ID
- Receive webhook notifications when payment confirms
- Query transaction status by order reference
- Access transaction history for reconciliation

<Note>
  API credentials and full technical documentation are provided after merchant onboarding. Contact the business team to get started.
</Note>
